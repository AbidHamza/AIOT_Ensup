# Guide de navigation - AIoT Academy

Ce guide vous aide à naviguer facilement dans le dépôt et à trouver rapidement ce dont vous avez besoin.

## Par où commencer ?

### Si vous débutez complètement

1. **Lisez le README principal** (`README.md`) : Il contient toutes les informations essentielles
2. **Commencez par la Room 0** : Elle pose les fondations théoriques
3. **Suivez les rooms dans l'ordre** : Chaque room prépare la suivante

### Si vous avez déjà de l'expérience

1. **Consultez la roadmap** dans le README principal
2. **Lisez les checklists** de chaque room pour voir ce que vous devez maîtriser
3. **Allez directement aux labs pratiques** si vous connaissez déjà la théorie

## Comment trouver ce que vous cherchez

### Je veux comprendre un concept théorique

→ Allez dans la room correspondante et lisez la section "Concepts abordés"

### Je veux faire un exercice pratique

→ Allez dans la room correspondante et cherchez la section "Labs" ou "Exercices"

### J'ai un problème technique

→ Consultez la section "Dépannage" du README principal ou le fichier `troubleshooting.md` de la room concernée

### Je veux soumettre mon travail

→ Consultez la section "Comment soumettre votre travail" dans le README principal

### Je cherche un fichier de configuration

→ Les fichiers de configuration sont généralement dans le dossier racine ou dans le dossier de la room concernée

## Navigation dans une room

Chaque room suit la même structure :

1. **Objectifs pédagogiques** : Ce que vous allez apprendre
2. **Concepts abordés** : La théorie expliquée simplement
3. **Prérequis** : Ce dont vous avez besoin avant de commencer
4. **Installation** : Comment configurer votre environnement
5. **Labs** : Exercices pratiques guidés
6. **Exercices** : Exercices à faire par vous-même
7. **Checklist de validation** : Pour vérifier vos acquis
8. **Dépannage** : Solutions aux problèmes courants
9. **Suite** : Vers quelle room passer ensuite

## Conseils de navigation

1. **Utilisez la recherche** : Si vous cherchez un terme spécifique, utilisez la fonction de recherche de votre éditeur
2. **Suivez les liens** : Les rooms contiennent des liens vers les autres rooms et ressources
3. **Consultez les README** : Chaque dossier important a un README.md qui explique son contenu
4. **Prenez des notes** : Créez votre propre fichier de notes pour suivre votre progression

## Fichiers importants à connaître

- **README.md** : Documentation principale (à lire en premier)
- **docker-compose.yml** : Configuration des services Docker
- **.gitignore** : Fichiers ignorés par Git (dont le dossier ressource)
- **CONTRIBUTING.md** : Guide de contribution (si vous voulez contribuer)

## Besoin d'aide ?

1. Consultez la section "Dépannage" du README principal
2. Vérifiez le README de la room concernée
3. Contactez votre formateur si le problème persiste
